# V2 深度 UX/UI 设计规范

> **来源**: 整合自 `docs/UX_LOGIC_V5.md` 与用户需求设计
> **版本**: V5.9 (Production Hardened)
> **核心原则**: 信任与流畅 (Trust & Flow) • 低摩擦复习 • 可解释同步

---

## 1. 设计哲学

在 V5.9 架构下，UI 不再只是数据的增删改查，而是一个 **"即时反馈的决策中心"**。

### 1.1 零延迟体验 (Zero-Latency)

| 原则 | 实现 |
|:---|:---|
| **Optimistic UI** | 点击 Rating 后立即预判下一张卡片，后端计算静默完成 |
| **预测文案** | 悬浮文案预测未来复习日期: `Good (in 4 days)` |
| **幂等保障** | `client_request_id` 确保网络重试不产生重复记录 |

### 1.2 系统级后悔药 (Atomic Undo)

| 场景 | UX 实现 |
|:---|:---|
| 合并标签 | 底部 Toast: "标签已合并 [撤销]" |
| 软删除学科 | 底部 Toast: "学科已删除 [撤销]" |
| 撤销时效 | 10 秒自动消失 |
| 后端支撑 | `undo_management_op` RPC |

## 2. 复习闭环设计 (Review Loop UX)

### 2.1 低摩擦原则

> **核心**: 复习闭环的摩擦是留存的最大敌人

| 原则 | 实现 |
|:---|:---|
| **评分按钮固定位置** | 底部固定 action bar，避免用户每题找按钮 |
| **作答输入可选** | 默认只需"看答案 → 评分"，高级用户可展开作答区 |
| **下一题切换 1-step** | 评分即进入下一题，无需二次确认 |
| **高密度键盘流** | `1/2/3/4` 直接评分、`Space` 显示答案 |

### 2.2 评分按钮语义

| rating | 数值 | 显示文案 | 快捷键 | 含义 |
|:---|:---|:---|:---|:---|
| Again | 1 | 重来 | `1` | 完全不记得，需要马上复习 |
| Hard | 2 | 困难 | `2` | 勉强记得，缩短间隔 |
| Good | 3 | 还行 | `3` | 正常回忆，正常间隔 |
| Easy | 4 | 轻松 | `4` | 毫不费力，延长间隔 |

题目是"资产"（可共享、可 fork），学习状态纯私有于用户。UI 必须把这层区分清楚。

## 7. 视觉识别系统

### 7.1 色彩体系

| 语义 | CSS 变量 | 用途 |
|:---|:---|:---|
| Primary | `--primary` | 主要操作、新卡片 |
| Accent | `--accent` | 强调、逾期提醒 |
| Warning | `--warning` | 到期提醒、Fork 更新 |
| Info | `--info` | 同步状态、提示 |
| Success | `--success` | 完成状态 |

### 7.2 状态语义 Token

```css
:root {
  /* 状态背景 */
  --state-due: rgba(233, 138, 21, 0.18);      /* 到期 */
  --state-overdue: rgba(217, 3, 104, 0.18);   /* 逾期 */
  --state-new: rgba(155, 197, 61, 0.14);      /* 新卡 */
  --state-sync: rgba(56, 189, 248, 0.14);     /* 有更新 */
  
  /* 键盘焦点 */
  --focus-ring: 0 0 0 3px rgba(var(--primary-rgb), 0.35);
}
```

### 7.3 微动效规范

| 场景 | 动效 |
|:---|:---|
| 评分按钮 hover | `translateY(-1px)` (轻微位移，非 scale) |
| 卡片切换 | 200ms fade + slide |
| 数字更新 | 平滑滚动动画 |
| 任务完成 | 热力图跳动 |

### 7.4 无障碍支持

```css
/* 键盘焦点必须可见 */
:focus-visible {
  outline: none;
  box-shadow: var(--focus-ring);
}

/* 尊重用户偏好 */
@media (prefers-reduced-motion: reduce) {
  .reveal { animation: none !important; }
  * { transition: none !important; }
}
```

## 8. 硬规则映射表

| 功能 | 后端逻辑 | 前端表现 |
|:---|:---|:---|
| Fork 归档题 | DB 拒绝 (`is_archived` check) | 按钮 Disabled + 悬浮提示 |
| 合并标签 | 原子合并 + 冲突去重 | 进度条 + 撤销气泡 |
| 权重修改 | 物理不可变触发器 | 提示"历史权重版本已锁定" |
| 软删除 | 索引条件隔离 | 列表消失 + 管理日志可恢复 |

## 9. 生物钟友好设计

### 9.1 日切换逻辑

基于 `review_settings.rollover_hour` 和 `timezone`:

```
用户设置: rollover_hour = 4, timezone = 'Asia/Shanghai'
→ 今日任务在 04:01 AM 刷新
→ 03:59 AM 完成的复习算"昨天"
```

### 9.2 UI 提醒

- 深夜 (22:00-06:00): 显示"夜间模式"指示
- 跨日边界: 显示"新的一天开始了"提示
- 连续天数: 基于 `last_study_day` 计算，显示在 Dashboard
