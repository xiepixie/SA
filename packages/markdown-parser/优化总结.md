# MarkdownRenderer 安全与体验优化总结

## 📋 核心改进概览

本次深度审计解决了所有 P0 级安全漏洞和 P1 级用户体验问题,同时完成了高质量的 UX/UI 设计优化。

---

## 🔒 P0 安全修复(已全部完成)

### 1. XSS 防护 - HTML 注入风险
**问题:** 后台题库内容/用户笔记通过 `innerHTML` 直接注入 DOM,存在脚本注入风险

**解决方案:**
- ✅ 增加 **DOMPurify** 多层防御
- ✅ 配置白名单:允许 KaTeX 所需的 `data-tex`、`data-key` 等属性
- ✅ 结合 Rust 端 `allow_dangerous_html: false`,实现纵深防御

### 2. LaTeX 注入防护
**问题:** KaTeX `\htmlData` 漏洞可注入恶意脚本

**解决方案:**
- ✅ 锁定 KaTeX 版本 ≥ 0.16.21(当前 **0.16.27**)
- ✅ 新增 `sanitizeLatex()` 函数,拦截危险命令:
  - `\htmlData`, `\HTML`, `\htmlClass`, `\htmlId`, `\htmlStyle`
- ✅ 渲染前过滤,安全日志记录

### 3. 无障碍访问修复被跳过
**问题:** `tabindex`/`role`/`aria-label` 仅在 `mathQueue.size > 0` 时设置,导致键盘用户无法操作

**解决方案:**
- ✅ 移至 `useLayoutEffect` 开头,**永远执行**
- ✅ 所有公式元素支持:
  - Tab 键聚焦
  - Enter/Space 键激活
  - 屏幕阅读器语义化("数学公式块,按 Enter 查看操作")

## 🎨 P1 用户体验优化(已全部完成)

### 1. 统一剪贴板 API
**问题:** 多处复制功能实现不一致,iOS/非 HTTPS 环境经常失败

**解决方案:**
- ✅ 全局使用 `copyToClipboard()` 降级策略:
  - 现代 Clipboard API → `execCommand` → 友好失败
- ✅ 可操作的错误提示:"复制失败。请长按选择文本手动复制"(对 iOS 友好)

**统一位置:**
1. 锚点链接复制
2. 代码块复制按钮
3. 公式菜单复制

### 2. 触屏/移动端支持
**问题:** 公式仅支持鼠标右键/双击,iPad/触摸屏完全无法使用

**解决方案:**
- ✅ **长按 350ms** 触发菜单(移动端标准)
- ✅ 触觉反馈(10ms 震动)
- ✅ 移动检测(10px 阈值)自动取消长按(防止滚动误触)
- ✅ 单击高亮,长按菜单(符合用户肌肉记忆)

### 3. 焦点管理
**问题:** 打开悬浮菜单后焦点停留在公式上,关闭时焦点丢失

**解决方案:**
- ✅ 打开时自动聚焦第一个按钮(复制)
- ✅ 关闭时焦点返回触发元素
- ✅ 符合 ARIA dialog/menu 模式

### 4. 性能优化 - 拆分渲染流程
**问题:** `useLayoutEffect` 中解析 Markdown 阻塞首帧绘制,长文档卡顿

**解决方案:**
- ✅ **步骤1 (useEffect):** 解析 + HTML 转换 → 非阻塞
- ✅ **步骤2 (useLayoutEffect):** morphdom 应用 DOM → 同步,无闪烁
- ✅ 兼顾性能与视觉稳定性

## 💡 额外优化

### 1. $ 符号识别修复
**问题:** `"$5"` 等货币符号被误识别为数学公式

**解决方案:**
- ✅ Rust 解析器检查真正的 LaTeX 定界符(`$$` 或成对的 `$...$`)
- ✅ 避免对货币金额的误报

### 2. 密集模式 CSS 完善
**问题:** `data-density="compact"` 仅部分实现

**解决方案:**
- ✅ 系统性收紧间距:
  - 段落:`margin: 0 0 0.5rem`
  - 列表:`margin: 0.5rem 0`
  - 标题:`margin: 1rem 0 0.5rem`
  - 数学块:`padding: 0.5rem 0.75rem`
- ✅ 适用于:侧边栏笔记、复习卡片、小屏设备
